Projekt 1: Server mit Php und Maria db installieren

Server Installieren

  1. sudo su

    2. Aktiviert root Rechte


  2. apt-get install apache2 php

    3. Installationsbefehl für die Pakete Apache2 und die aktuellste Version von Php


  3. usermod -a -G www-data username

    4. Fügt deinen user zur Gruppe www-data hinzu


  4. chmod g+wx /var/www/html

    5. Gibt der Gruppe des Verzeichnisses /var/www/html die Schreib und Ausführberechtigung


  5. chown username:www-data /var/www/html

    6. Ändert die Gruppe des Verzeichnisses /var/www/html zu www-data.
    /var/www/html ist das Verzeichnis in dem die Dateien für Webseite gespeichert werden müssen


    Datenbank installieren


  6. apt-get install mariadb-server

    7. Installationsbefehl für das Paket mariadb-server


  7. mysql_secure_installation

    8. Startet die Installation der Datenbank


  8. Passwort erstellen Fragen beantworten
  9. mysql -u root -p

    10. Admin Login in die Konsole der Datenbank


  10. CREATE DATABASE usernamedb;

    11. Erstellt eine Datenbank mit Namen Usernamedb


  11. CREATE USER ‘username‘@’localhost’ IDENTIFIED BY ‘passwort‘;

    12. Erstellt User mit Namen Username und Passwort


  12. GRANT ALL PRIVILEGES ON *.* TO ‘username‘@’localhost’ WITH GRANT OPTION;

    13. Vergibt alle Rechte inklusive dem Recht Rechte zu vergeben an den User Username


  13. FLUSH PRIVILEGES;

    14. Speichert die Änderungen


  14. QUIT

    15. Logout aus der Datenbank


    Grafische Oberfläche


  15. apt-get install phpmyadmin

    16. Installationsbefehl für das Paket Php my admin


  16. nano /etc/apache2/apache2.conf
    Füge am Ende “Include /etc/phpmyadmin/apache.conf” ein

    17. Öffnet das Dokument apache2.conf im Editor nano


  17. systemctl restart apache2

    18. Starte den Server neu
    Die Oberfläche kann über den Browser unter hostname.local/phpmyadmin erreicht werden

Projekt 2: Das CMS-System Bludit

  1. sudo su

    2. Aktiviert Root Rechte


  2. cd /var/www/html

    3. Öffnet das Verzeichnis /var/www/html


  3. git clone https://github.com/jeremehancock/Bludit-Pi.git

    4. Klont das Github Repository Bludit-Pi in das geöffnete Verzeichnis


  4. cd Bludit-Pi

    5. Öffnet das Verzeichnis Bludit-Pi


  5. bash build.sh

    6. Führt die Installation von Bludit aus


  6. nano /etc/apache2/apache2.conf

    7. Öffnet die Datei apache2.conf im Editor Nano


  7. Ändere die Datei wie folgt, um htaccess zu aktivieren:


    Options FollowSymLinks
    AllowOverride All
    Require all granted


    Options Indexes FollowSymLinks
    AllowOverride All
    Require all granted

    8. Diese Änderung aktivieren ht access auf deinem Server


  8. systemctl restart

    9. Starte deinen Pi neu

  9. nano apache2/sites-available/bludit.conf Verzeichnis auf /var/www/html/ ändern

    10. Ändert das Verzeichnis des Servers auf /var/www/html

  10. systemctl restart apache2

    11. Startet den Server neu
